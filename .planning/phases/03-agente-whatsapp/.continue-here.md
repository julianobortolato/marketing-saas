# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure in this session.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: GRANTS_WITH_TABLE — toda migration CREATE TABLE deve incluir `GRANT SELECT/UPDATE/DELETE ON public.tabela TO authenticated; GRANT ALL ON public.tabela TO service_role;` na MESMA migration. Esquecer causa "permission denied" ao testar no browser, mesmo com RLS correto.
- [ ] CONSTRAINT: SUPABASE_JOIN_TYPESCRIPT — joins via `.select('tabela_relacionada(coluna)')` retornam `tabela_relacionada: []` (array), não objeto — TypeScript recusa cast direto; usar `as unknown as T`.
- [ ] CONSTRAINT: RPC_SIGNATURE_FIRST — antes de escrever qualquer chamada RPC, ler a migration da função para verificar a assinatura exata. Plans podem descrever estado futuro; migration tem o estado real.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| GRANTS_MISSING | Migration 0007 criou conversas/chat_messages sem GRANTs → "permission denied for table conversas" ao testar | blocking | Toda migration CREATE TABLE inclui GRANT na mesma migration |
| RPC_SIGNATURE_DRIFT | Plan 03-02 omitiu p_tenant_id da chamada rpc_handoff_humano; RPC exige 3 params | advisory | Ler migration da função antes de escrever caller |
| SUPABASE_JOIN_ARRAY | Joins Supabase retornam array mesmo em FK many-to-one; build quebra com cast direto | advisory | Cast via `as unknown as T` para todos os joins |
