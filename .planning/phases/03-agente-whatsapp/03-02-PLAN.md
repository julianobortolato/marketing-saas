Purpose: This plan delivers WHATS-03 (manual handoff via dashboard toggle) end-to-end and makes the agent operable in production. The diagnostics endpoint also unblocks the Sprint 4 cutover monitoring. Editorial CRUD activates the academia_config columns added in migration 0006 so the agent system prompt actually reads tenant-configured editorial content (not stub defaults).

Output: Three new Server Component pages (conversas list, conversa detail, editorial config), one diagnostics API route, two Server Actions (handoff + editorial save), and an updated sidebar with a Conversas nav item.
</objective>
