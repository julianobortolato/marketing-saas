<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser -> Next.js middleware | Every request carries (possibly forged/expired) auth cookie; validated against Supabase auth server |
| Next.js Server -> Supabase Postgres | Server clients use anon key + RLS; admin client (service role) is server-only |
| Local dev -> Supabase project | Migrations and secrets cross here; service role key must never enter client bundle or git |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Information Disclosure | All tables (cross-tenant read) | mitigate | RESTRICTIVE RLS policy on tenants/usuarios/academia_config in migration 0004; `(SELECT fn_tenant_id())` boundary |
| T-01-02 | Tampering | fn_tenant_id / fn_usuario_role | mitigate | SECURITY DEFINER functions read from `public.usuarios`, never `auth.jwt()` (CLAUDE.md anti-pattern enforced) |
| T-01-03 | Elevation of Privilege | Service role key exposure | mitigate | `lib/supabase/admin.ts` first line `import 'server-only'`; server.ts uses anon key only (grep gate in Task 2) |
| T-01-05 | Spoofing / Auth Bypass | Stale or forged session in middleware | mitigate | `supabase.auth.getUser()` (hits auth server), `getSession()` forbidden — grep gate in Task 2 |
| T-01-SC | Tampering | npm dependency installs | mitigate | All packages [Approved] in RESEARCH § Package Legitimacy Audit (npm-registry verified); no [ASSUMED]/[SUS]/[SLOP] packages — no blocking checkpoint required; zod pinned to 3.25.76 |
