## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser form -> Server Action | Client form payload is untrusted; revalidated with zod server-side |
| Server Action -> academia_config | tenant_id must come from fn_tenant_id(), never the client |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Tampering | tenant_id in upsert payload | mitigate | tenant_id resolved server-side via `supabase.rpc('fn_tenant_id')`; RLS WITH CHECK rejects mismatched tenant_id (Plan 01 migration 0004) |
| T-01-01 | Information Disclosure | academia_config cross-tenant read | mitigate | RESTRICTIVE RLS from Plan 01; query relies on RLS scoping, no client tenant_id |
| T-01-10 | Tampering | Client form input | mitigate | zod safeParse server-side in the Server Action (not just client) |
| T-01-11 | Denial of Service | Silent NULL tenant_id (Pitfall 7) | mitigate | Explicit error returned, never a silent partial write |
| T-01-SC | Tampering | npm installs | accept | No new packages (all from Plan 01 Approved set) |
</threat_model>
