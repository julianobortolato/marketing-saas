| Token | Value | Usage |

## Interaction Contracts

### Form validation

- Validation fires on `onBlur` for each field (not on every keystroke)
- Required fields show error on blur if empty: red border (`#E30613` ‚Üí use destructive `#DC2626` for errors), red helper text 14px below input
- Form submit button disabled until all required fields (`*`) are non-empty
- No full-page validation summary ‚Äî errors are inline, field-level only

### Loading states

- All async actions (signup, login, save config): button shows `Loader2` spinner + descriptive text, disabled, no other UI change
- Duration expectation: Supabase calls < 2s ‚Äî no skeleton needed for Phase 1 forms

### Auth redirect rules

| Condition | Redirect |
|-----------|----------|
| Logged out, visits `/dashboard/*` | ‚Üí `/login` |
| Logged in, visits `/login` or `/signup` | ‚Üí `/dashboard/overview` |
| First signup (new tenant, no `academia_config`) | ‚Üí `/dashboard/configuracoes` |
| Returning login (config exists) | ‚Üí `/dashboard/overview` |

### Role-based UI rules

| Role | Can see | Cannot see |
|------|---------|------------|
| `owner` | All sections, all CTAs | ‚Äî |
| `manager` | All sections, all CTAs | Billing fields (deferred to future phase) |
| `viewer` | All sections read-only | Save/submit buttons hidden (not disabled ‚Äî hidden) |

Viewer enforcement: form fields use `readOnly` attribute, save button conditionally not rendered (`role !== 'viewer'`).

### Empty states

| Screen | Empty State |
|--------|-------------|
| `/dashboard/overview` ‚Äî no DNA configured | Welcome card with "Configurar agora" CTA (see screen inventory above) |
| `/dashboard/overview` ‚Äî DNA configured, no leads yet | 3 placeholder counter cards showing "0" with muted labels |
| `/dashboard/configuracoes` ‚Äî first load | Pre-filled with defaults: raio_km = 5, tom_de_voz = "neutro" |

## Accessibility Minimums

- All form inputs have associated `<label>` via `htmlFor` / `id` (not `aria-label` only)
- Error messages linked to inputs via `aria-describedby`
- Password toggle button has `aria-label="Mostrar senha"` / `"Ocultar senha"`
- Color alone never conveys state ‚Äî error inputs have both red border AND error text
- Nav items have `aria-current="page"` when active
- All interactive elements keyboard-navigable (shadcn components satisfy this via Radix)
